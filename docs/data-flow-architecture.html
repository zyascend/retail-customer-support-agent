Retail Customer Support Agent — Data Flow

端到端数据流 · 从用户输入到数据库变更 · 覆盖 7 层安全守卫

核心设计原则

  • • LLM 不决策写操作是否允许——只负责调用工具
  • • Guard 层是写操作的唯一决策者
  • • 过早拒绝安全网强制注入 write tool call
  • • action_specs.py 是写操作的单一事实来源

7 层守卫阻断统计

  • • authentication_required — 用户未登录
  • • explicit_confirmation_required — 需要用户确认
  • • ownership_violation — 非本人订单
  • • non_pending/delivered — 订单状态不符
  • • replacement_item_* — 商品替换问题
  • • payment_method_* — 支付方式/余额问题
  • • duplicate_write_lock — 并发操作冲突

运行时指标

  • • AgentLoop 最大迭代: 14 次
  • • 连续工具失败上限: 3 次
  • • Auto-load 重试上限: 1 次
  • • 会话消息保留: 最近 6 条
  • • ContextBuilder 预算: ~1200 tokens
